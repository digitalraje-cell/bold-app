# Bold v0.2.0 Release Notes

**Release date:** 2026-06-22  
**Branch:** `release/v0.2.0`  
**Tag:** `v0.2.0`

## Overview

Bold v0.2.0 focuses on production meeting reliability, host/participant UX, mobile polish, and removing all Jitsi authentication from the user experience. Bold is now the sole authentication and role layer; media access is issued via JWT when self-hosted Jitsi is configured.

---

## Meeting stability improvements

- Jitsi auto-reconnect with iframe navigation guard (blocks 8x8/promo/auth redirects)
- Real-time participant join/leave via Socket.io (`participant:joined`, guest leave endpoint)
- Host-first media gating fallback when JWT is not configured (avoids public Jitsi moderator screens)
- Production DB schema sync on deploy (`migrate deploy` + `db push` fallback)
- Dashboard Prisma queries fixed after schema migration (`durationMinutes`, `registrationRequired`, `meeting_registrants`)

## Co-host management

- Promote/remove co-host via participants panel API
- Co-host treated as moderator in Bold UI (mute all, invite, room mode, chat mode)
- JWT `moderator: true` issued for Host and Co-host roles

## Audio / video sync fixes

- Join media preferences persisted (`readJoinMediaPrefs`) — start muted/video-off respected in room
- Webinar mode auto-mutes participants off-stage when mic/camera not allowed
- Host `host:media-ready` / `host:media-left` socket events keep guest media timing in sync

## Join without audio / video

- Lobby device toggles for mic/camera before join
- Jitsi embed honors `startWithAudioMuted` / `startWithVideoMuted` from join prefs
- Prejoin page disabled — users enter room directly from Bold lobby

## Mobile UI improvements

- Controls bar overflow → core actions + **More** menu for secondary controls
- Full-screen participant panel on mobile
- Sticky toolbar and Jitsi shell min-height for small viewports
- Guest waiting screen and media error states optimized for mobile

## Jitsi authentication removal

- All Jitsi login, moderator login, lobby auth, and welcome/prejoin pages disabled in embed config
- `loginRequired` and external auth URL navigation show **Bold error UI** — never Jitsi login screens
- Bold branding only (`APP_NAME: Bold`, watermarks and promos hidden)
- New `MediaConnectionError` component with retry and back-to-lobby actions

## JWT meeting access flow

- New API: `POST /meetings/:id/jitsi-token`
- Validates admitted Bold participant before issuing token
- Host / Co-host → JWT with `moderator: true`
- Guest participants → JWT with `moderator: false` (no login required)
- Configure on API: `JITSI_DOMAIN`, `JITSI_APP_ID`, `JITSI_APP_SECRET`
- When JWT not configured, legacy open embed with host-first gating (dev / public Jitsi fallback)

- Scheduled meeting duration + end-time preview
- Password vs direct-link join behavior
- **Join Meeting Again** for returning guests
- Toolbar, lobby, and guest label fixes from post-production pass
